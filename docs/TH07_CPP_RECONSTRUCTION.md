# TH07 C++ Reconstruction Notes

`src/` is now a flat TH06/GensokyoClub-style source tree. It is not a binary-perfect target and it is not the final modern engine; it is the source-level workspace where TH06-derived code is replaced by TH07 evidence in place.

Portable, modernized gameplay helpers live in the same flat `src/` tree, while deterministic gameplay code still must stay detached from SDL, Vulkan, Win32, and D3D8 assumptions.

Current evidence roots:

- Ghidra string export: `reference/ghidra/strings.tsv`
- TH07 extracted archive: `reference/extracted/`
- readable ECL/STD/MSG: `reference/ecl/`, `reference/std/`, `reference/msg/`

## Bomb Evidence Notes

`FUN_004423e0` copies four routine pointers from `PTR_FUN_0049ec50` into player fields. The original executable calls `0x16a3c`/`0x16a44` from the update path and `0x16a40`/`0x16a48` from the draw path; `0x16a24`, copied from the focus flag when the bomb starts, selects unfocused versus focused slots. The startup path writes `0x16a20 = 1`, initializes the bomb timer fields at `0x16a34`/`0x16a38`, seeds `0x16a28` to 999 before the routine-specific calc runs, and increments the player power counter at `0x23f8` by 6 frames/items before clamping to the current shot-data max.

Confirmed bomb routine start constants:

`FUN_004429d0` proves the original global player object base and reset span: it clears `DAT_004bdad8` for `0x2df9e` dwords (`0x0b7e78` bytes), then seeds the common effect fields at `+0x16a00`, `+0x16a04`, and `+0x16a08`. Those offsets feed the common bomb-start effect helper `FUN_004084f0`; the third value is the common effect duration, not the movement state timer. `BuildPlayerBombCommonEffectConfig` records the source-derived spawn contract: effect id `0x19`, effect table entry `0x0049efc0 + 0x19 * 0x0c`, ANM script `0x02da`, per-frame callback `FUN_0041c1b0`, initial color `0xffffffff`, tint `0xffff4040`, interpolation scale `0x3d800000`, and the previous active effect pointer at player offset `0x0b7e68`. This is also an important TH07-vs-TH06 effect-system gap: the copied TH06 effect table shape is not authoritative for bomb runtime replacement.

The first player layout gap is also now explicit. Original TH07 player rendering uses the same `0x24c` ANM VM slot size already proven for Stage, not the copied TH06 `AnmVm` size `0x110`. `FUN_00441fb0` executes the primary player VM at `+0x0000` and the option VMs at `+0x024c` and `+0x0498`; `FUN_004420b0` draws those same option VMs and writes their draw positions at `+0x0414` and `+0x0660` from option positions `+0x09b4` and `+0x09c0`. `FUN_004423e0` initializes primary script `0x0400` at `+0x01d8`, left option script `0x0480` at `+0x0424`, right option script `0x0481` at `+0x0670`, and player position at `+0x0930`. `src/PlayerLayout.hpp` records `0x0930 == 4 * 0x024c` as the original ANM VM prefix size, and `Player.hpp` now uses `Th07PlayerOriginalAnmVmSlot` for the four leading VM slots while preserving the old `AnmVm` call surface through inheritance. This is a structural bridge, not full behavior parity: the runtime still calls the TH06-shaped update/draw functions and the original TH07 ANM VM executor is not yet wired for player VMs.

The next player field slice is anchored around the same original runtime. `FUN_004423e0` seeds position `+0x0930/+0x0934/+0x0938`, option positions `+0x09b4/+0x09c0`, hitbox size `+0x0990`, graze box size `+0x099c`, and grab-item box size `+0x09a8`. `FUN_0043ee50` writes movement deltas at `+0x09cc/+0x09d0`, clamps position, and derives three min/max boxes from position plus those size vectors: kill hitbox `+0x0948/+0x0954`, graze box `+0x0960/+0x096c`, and grab-item box `+0x0978/+0x0984`. `FUN_0043e260`, `FUN_0043e3b0`, `FUN_0043e4e0`, and `FUN_0043e6b0` consume those fields for bullet kill, graze, item-box, and laser collision checks. `Player.hpp` now statically asserts those offsets through `+0x09d0`, and the known bomb speed multiplier fields are padded to `+0x23f0/+0x23f4`. Exact SHT-derived size initialization, graze behavior, item behavior, and the rest of the player object remain pending.

The mode/option owner slice is now separated from the old TH06-shaped member names. `FUN_004423e0` initializes raw mode state `+0x2408 = 1`, raw option state `+0x240a = 1`, and common-effect defaults `+0x16a00 = -999`, `+0x16a04 = 0`, `+0x16a08 = 0x78`. `FUN_0043ee50` proves `+0x240a` is the option/focus visual state (`0..4` = hidden/unfocused/focusing/focused/unfocusing), `+0x240b` is the focus-held byte, and `+0x2410/+0x2414/+0x2418` are the option interpolation timer fields. `src/Player.cpp` now applies only the low-byte initial mode/option/request state through `BuildPlayerInitialModeOwnerState`; the common-effect tail and active effect pointers remain metadata-only because the live `Player` structure still does not carry the original TH07 object tail.

Bomb movement suppression/boosting is written through `0x23f0` and `0x23f4`, which `FUN_0043ee50` multiplies into horizontal and vertical movement. `src/PlayerBombTables.*` also records the exact PE addresses for each calc routine's duration write, common-effect duration write, minimum score-drain load, and score-drain-factor push before `FUN_00408610`; `tests/test_sht_parser.cpp` checks those instruction immediates against `reference/original/th07.exe`.

`FUN_00408610` is now represented by `ComputePlayerBombScoreDrainStep`: score delta is multiplied by the routine's factor, scaled by difficulty (`Hard / 2`, `Lunatic / 4`, `Extra`/`Phantasm / 3`), divided by bomb duration, rounded down to a multiple of 10, and clamped up to the routine minimum divided/rounded the same way.

`FUN_004083f0` is represented by `ComputePlayerBombBackdropColor` and now wired into the current bomb draw runtime through `BombData::DarkenViewport`. The helper derives the bomb backdrop draw color from `0x16a38` timer frames and `0x16a28` duration: 60-frame fade in, plateau at `0x80303030`, and 60-frame fade out, always with alpha `0x80`. The runtime now resets the ANM manager color multiplier, calls the stage color writer equivalent `ApplyTh07StageAnmManagerColorWrite`, and sets the stage color reset guard. The original `_DAT_0134cdb0 = 1` write is `DAT_01347b00 + 0x52b0`, matching `Stage::th07AnmManagerColorResetGuard`; this replaces the previous TH06 rectangle-darken path for this producer.

The remaining player-side `FUN_00406930` producers are now mapped instead of left as a renderer mystery. `FUN_00441330` calls the stage color writer at `0044162f` while player state `+0x2408` is mode `4` and common-effect duration `+0x16a08` is positive; `FUN_004420b0` repeats the same write at `0044233b` from the draw chain under the same state/duration condition. Both read `DAT_004d44e0`, which `FUN_004429d0` proves is the global-player absolute alias for `DAT_004bdad8 + 0x16a08`, with a 30-frame fade in, plateau through frame `509`, fade out from frame `510` toward total frame `540`, delta `0x50`, plateau channel `0x30`, and alpha `0x80`; `ComputePlayerMode4StageColor` records that source-derived formula. The full player update/draw runtime is still pending, so these helpers are evidence-backed data and formula coverage rather than a claim that mode `4` behavior is fully replay-exact.

The mode transition effect metadata is now anchored as a separate source-derived slice. `FUN_00441960` enters raw mode `4` by writing `+0x16a08 = 0x21c`, mirroring the common effect vector into `+0x16a0c..+0x16a14`, writing transition request `+0x240d = 1`, writing mode state `+0x2408 = 4`, killing the old pointers at `+0x0b7e6c` and `+0x0b7e68`, and spawning effect id `0x1c` from the player position at `+0x930` with slot argument `4`, flag `1`, color `0xffffffff`, scale bits `0x3f800000 -> 0x3e800000`, and active pointer `+0x0b7e6c`. `FUN_00441bd0` and `FUN_00441670` transition raw mode `2/4` into raw mode `3`: they write request `+0x240d = 0`, mode state `+0x2408 = 3`, common effect duration `+0x16a08 = 0x28`, lockout `+0x23fc = 0x28`, and the same active pointer; the `FUN_00441bd0` effect interpolation slice uses `0x1e` frames and scale bits `0x3d800000 -> 0x3fa66666`. `FUN_00441670` also contains a mode-1 cleanup write of `+0x16a08 = 0xf0` before the unconditional mode-3 write overwrites the common effect duration. These writes are now exposed as `BuildPlayerMode4EntryOwnerState` and `BuildPlayerMode3EntryOwnerState` metadata, while active-pointer killing/spawning and mode-4 color producers still wait for a proven TH07-sized Player owner layout.
